This ProgressBar is honestly already solid — you’ve got the NOW pill + pin, progress fill, clamping, past-target, beyond-stop. The “unclear” feeling is not marker math; it’s information hierarchy + consistency + a couple edge cases.

Here are the changes I’d make (all small, high impact).

⸻

1) Biggest fix: stop using “guessed entry/stop” in Trade Now

Right now, if entryPriceAtActivation or stopPrice is null, you fabricate entry/stop from magnet +/- stopDistance*2. That can easily produce nonsense, and it’s why the UI can feel “historical / theoretical.”

Trade Now cards should only render if the trade is truly ACTIVE, meaning:
	•	activationStatus === "ACTIVE"
	•	entryPriceAtActivation != null
	•	stopPrice != null

If any of those are missing, show a small badge: “Live trade data incomplete” and don’t show the bar.

Change inside ProgressBar:
Replace the computed entry/stop fallback with “require actual values” for Trade Now.

const entryPrice = signal.entryPriceAtActivation;
const stopPrice = signal.stopPrice;
if (entryPrice == null || stopPrice == null) return null;

That immediately increases trust.

(You can keep fallback logic for OnDeck cards, but not Trade Now.)

⸻

2) Fix scale distortion: don’t let NOW expand the axis range

You currently include currentPrice in allPrices. That’s fine, but it causes the bar to “re-scale” as price moves, which makes the markers drift and can feel confusing.

Better behavior for traders:
Scale should be stable: min/max should be based on stop and target (and maybe entry), not current.

Change:

const allPrices = [magnetPrice, entryPrice, stopPrice]; // no currentPrice
const priceMin = Math.min(...allPrices);
const priceMax = Math.max(...allPrices);

Then compute rawCurrentPct against that stable range. You already clamp and mark currentClamped, so it still works perfectly.

This makes the progress bar feel like “a map” instead of “rubber band.”

⸻

3) Make the direction obvious: put Stop on the “risk side,” Target on the “reward side”

Right now stop/entry/target can appear left-to-right in any order depending on numbers. Traders read BUY as: Stop (left) → Entry → Target (right), and SELL as reverse.

Since you’re already normalizing, you can label the endpoints:

Add 2 tiny labels above the bar ends:
	•	Left end: “RISK” (for BUY) or “TARGET” (for SELL)
	•	Right end: “TARGET” (for BUY) or “RISK” (for SELL)

This is a pure UX cue—no math change.

⸻

4) Use tp.t1 consistently (don’t mix magnetPrice and tp.t1)

You’re using magnetPrice as target marker and tooltip, while the card shows tp.t1. These must match or it looks buggy.

Change:

const t1 = tp.t1 ?? signal.magnetPrice;
const magnetPrice = t1;

Then everything references the same number.

⸻

5) Show “+/- from entry” inside the NOW pill (tiny, huge value)

Right now NOW pill shows price only. Add delta:

For BUY:
	•	Δ = currentPrice - entryPrice
For SELL:
	•	Δ = entryPrice - currentPrice

Display as:
	•	NOW 685.47 (+0.59)
and color is already handled by isWinning.

This makes the “where am I now” instantly readable.

⸻

6) Add a second mini-line: “R now” in the NOW pill (optional)

If stop exists, compute rNow locally (even if server doesn’t provide it yet):

BUY:
	•	(currentPrice - entryPrice) / (entryPrice - stopPrice)
SELL:
	•	(entryPrice - currentPrice) / (stopPrice - entryPrice)

Then append:
	•	• 0.27R

That replaces needing another row for performance.

⸻

7) Pace label: use it on the card, not the bar

Your bar is already dense. Keep pace label in Row 5 near Early Hit:
	•	“On pace” or “Late” pill

The bar should stay visual.

⸻

Drop-in revised ProgressBar (minimal diff)

Here’s a version that implements the key improvements: stable scale, require real entry/stop for Trade Now, and consistent t1.

function ProgressBar({ signal, currentPrice }: { signal: SignalApi; currentPrice?: number }) {
  const tp = signal.tradePlanJson as TradePlan | null;
  if (!tp) return null;

  // Use a single target source of truth
  const targetPrice = (tp.t1 ?? signal.magnetPrice);

  // For TradeNow (ACTIVE), require real entry/stop for trust
  const entryPrice = signal.entryPriceAtActivation ?? null;
  const stopPrice = signal.stopPrice ?? null;
  if (signal.activationStatus === "ACTIVE" && (entryPrice == null || stopPrice == null)) return null;

  // For non-active views, you can keep fallbacks (optional)
  const resolvedEntry =
    entryPrice ??
    signal.entryTriggerPrice ??
    (tp.bias === "SELL"
      ? targetPrice + (tp.stopDistance ?? 0) * 2
      : targetPrice - (tp.stopDistance ?? 0) * 2);

  const resolvedStop =
    stopPrice ??
    (tp.bias === "SELL"
      ? resolvedEntry + (tp.stopDistance ?? 0)
      : resolvedEntry - (tp.stopDistance ?? 0));

  const isSell = tp.bias === "SELL";

  // STABLE SCALE (do not include currentPrice)
  const priceMin = Math.min(targetPrice, resolvedEntry, resolvedStop);
  const priceMax = Math.max(targetPrice, resolvedEntry, resolvedStop);
  const range = priceMax - priceMin || 1;

  const toPercent = (price: number) => Math.max(0, Math.min(100, ((price - priceMin) / range) * 100));

  const entryPct = toPercent(resolvedEntry);
  const targetPct = toPercent(targetPrice);
  const stopPct = toPercent(resolvedStop);

  const rawCurrentPct = currentPrice != null ? ((currentPrice - priceMin) / range) * 100 : null;
  const currentPct = rawCurrentPct != null ? Math.max(0, Math.min(100, rawCurrentPct)) : null;
  const currentClamped = rawCurrentPct != null && (rawCurrentPct < 0 || rawCurrentPct > 100);

  const beyondStop = currentPrice != null && ((isSell && currentPrice > resolvedStop) || (!isSell && currentPrice < resolvedStop));
  const pastTarget = currentPrice != null && ((isSell && currentPrice < targetPrice) || (!isSell && currentPrice > targetPrice));

  const progressFillLeft = currentPct != null ? Math.min(entryPct, currentPct) : null;
  const progressFillWidth = currentPct != null ? Math.abs(currentPct - entryPct) : null;

  const isWinning =
    currentPrice != null && ((!isSell && currentPrice > resolvedEntry) || (isSell && currentPrice < resolvedEntry));

  const nowPillAnchor =
    currentPct != null ? (currentPct > 85 ? "right" : currentPct < 15 ? "left" : "center") : "center";

  const delta =
    currentPrice != null ? (isSell ? (resolvedEntry - currentPrice) : (currentPrice - resolvedEntry)) : null;

  return (
    <div className="space-y-0.5" data-testid={`progress-bar-${signal.id}`}>
      <div className="relative h-4 pt-5">
        {currentPct != null && currentPrice != null && (
          <div
            className="absolute z-30 flex flex-col items-center"
            style={{
              left: `${currentPct}%`,
              transform:
                nowPillAnchor === "center" ? "translateX(-50%)" : nowPillAnchor === "right" ? "translateX(-90%)" : "translateX(-10%)",
              top: 0,
            }}
            data-testid={`marker-now-${signal.id}`}
          >
            <span
              className={`text-[10px] font-bold px-1.5 py-0.5 rounded-full whitespace-nowrap leading-none ${
                beyondStop
                  ? "bg-red-500 text-white animate-pulse"
                  : pastTarget
                    ? "bg-emerald-500 text-white"
                    : isWinning
                      ? "bg-emerald-500/90 text-white"
                      : "bg-amber-500/90 text-white"
              }`}
              data-testid={`text-now-price-${signal.id}`}
            >
              NOW {currentPrice.toFixed(2)}
              {delta != null && ` (${delta >= 0 ? "+" : ""}${delta.toFixed(2)})`}
              {currentClamped && " ⚠"}
              {pastTarget && " Past T1"}
            </span>
            <div className={`w-0.5 h-2.5 ${beyondStop ? "bg-red-500" : isWinning ? "bg-emerald-500" : "bg-amber-500"}`} />
          </div>
        )}

        <div className="relative h-3 rounded-full bg-muted overflow-hidden">
          {progressFillLeft != null && progressFillWidth != null && progressFillWidth > 0 && (
            <div
              className={`absolute h-full rounded-full transition-all duration-300 ${
                isWinning ? "bg-emerald-400/50 dark:bg-emerald-500/40" : "bg-red-400/50 dark:bg-red-500/40"
              }`}
              style={{ left: `${progressFillLeft}%`, width: `${progressFillWidth}%` }}
              data-testid={`fill-progress-${signal.id}`}
            />
          )}

          <div
            className="absolute w-1.5 h-full rounded-sm bg-red-400/70 dark:bg-red-400/60"
            style={{ left: `${stopPct}%`, transform: "translateX(-50%)" }}
            title={`Stop: ${resolvedStop.toFixed(2)}`}
            data-testid={`marker-stop-${signal.id}`}
          />

          <div
            className="absolute w-2.5 h-2.5 rounded-full bg-muted-foreground/80 border-2 border-background top-[1px]"
            style={{ left: `${entryPct}%`, transform: "translateX(-50%)" }}
            title={`Entry: ${resolvedEntry.toFixed(2)}`}
            data-testid={`marker-entry-${signal.id}`}
          />

          <div
            className={`absolute w-2.5 h-2.5 rounded-full top-[1px] border-2 border-background ${isSell ? "bg-red-500" : "bg-emerald-500"}`}
            style={{ left: `${targetPct}%`, transform: "translateX(-50%)" }}
            title={`T1: ${targetPrice.toFixed(2)}`}
            data-testid={`marker-target-${signal.id}`}
          />

          {currentPct != null && (
            <div
              className={`absolute w-0.5 h-full z-20 ${beyondStop ? "bg-red-500" : isWinning ? "bg-emerald-500" : "bg-amber-500"}`}
              style={{ left: `${currentPct}%`, transform: "translateX(-50%)" }}
            />
          )}
        </div>
      </div>

      <div className="flex justify-between text-[10px] text-muted-foreground px-0.5">
        <span title={`Stop: ${resolvedStop.toFixed(2)}`}>Stop {resolvedStop.toFixed(2)}</span>
        <span title={`Entry: ${resolvedEntry.toFixed(2)}`}>Entry {resolvedEntry.toFixed(2)}</span>
        <span title={`T1: ${targetPrice.toFixed(2)}`}>T1 {targetPrice.toFixed(2)}</span>
      </div>
    </div>
  );
}

What this accomplishes

	•	Trustworthy Trade Now (no guessed entries/stops)
	•	Consistent target (tp.t1 and marker align)
	•	NOW pill tells you “where am I vs entry” instantly
